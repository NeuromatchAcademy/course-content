# 2026 Day Renaming Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Rename tutorial days to reflect the 2026 schedule shift (new W2D2 inserted, all downstream days shifted +1, OptimalControl demoted to Bonus) and create a stub W2D2 placeholder directory.

**Architecture:** Each task operates on one logical unit (one day or one config file). All renames use `git mv` so history is preserved. Generated dirs (`student/`, `instructor/`, `solutions/`, `static/`) are deleted with `git rm -r` — the processing pipeline recreates them with correct names on the next PR. The final task verifies the complete state.

**Tech Stack:** git, Python (for `materials.yml` editing), bash.
